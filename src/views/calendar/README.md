# 日历功能模块说明

## 功能概述

这是一个基于 VCalendar + Element UI 2.x 开发的日历功能模块，提供完整的日程管理和日志记录功能。

## 主要特性

### 1. UI 布局核心
- **月份概览页**：使用 Element UI 的 el-row/el-col 栅格组件实现三行四列布局
- **月份详情页**：通过 el-container/el-header/el-main 搭建页面框架
- **日志录入**：使用 el-dialog 弹窗承载 el-form 表单
- **统一交互**：所有组件均使用 Element UI 原生组件，保证样式一致性

### 2. 核心功能
- **月份浏览**：直观的12个月份卡片展示
- **日历视图**：集成 VCalendar 组件提供专业的日历界面
- **日志管理**：支持添加、编辑、查看日志记录
- **数据持久化**：使用 Vuex + localStorage 实现数据存储
- **响应式设计**：适配不同屏幕尺寸

## 使用方法

### 1. 访问日历
点击顶部导航栏的"日历"选项即可进入日历概览页面。

### 2. 查看月份详情
在概览页面点击任意月份卡片，即可跳转到该月份的详细视图。

### 3. 添加日志
在月份详情页：
- 点击日历中的任意日期
- 在弹出的对话框中填写日志内容
- 点击"确定"保存日志

### 4. 编辑日志
- 点击已有日志的日期
- 修改日志内容
- 保存更改

## 数据结构

### 日志对象格式
```javascript
{
  id: 123456789,           // 唯一标识符
  date: '2024-01-15',      // 日期 (YYYY-MM-DD)
  content: '日志内容...',   // 日志文本内容
  timestamp: '2024-01-15T10:30:00.000Z' // 创建时间
}
```

### 存储结构
```javascript
{
  '2024-01': [log1, log2, ...],  // 一月份的日志数组
  '2024-02': [log3, log4, ...],  // 二月份的日志数组
  ...
}
```

## 样式规范

### 设计原则
- 遵循 Element UI 设计规范
- 使用渐变色作为主要视觉元素
- 保持简洁、现代的界面风格
- 注重用户体验和交互反馈

### 响应式断点
- 桌面端：≥ 768px
- 移动端：< 768px

## 注意事项

1. **依赖要求**：需要安装 `v-calendar@next` 依赖包
2. **浏览器兼容**：支持现代浏览器（Chrome、Firefox、Safari、Edge）
3. **数据存储**：数据存储在 localStorage 中，清除浏览器数据会丢失日志
4. **性能优化**：大量日志数据可能影响页面性能，建议定期清理

## 开发扩展

### 添加新功能
1. 在对应的 Vue 组件中添加新的模板和逻辑
2. 更新 Vuex store 以支持新的数据操作
3. 修改路由配置以支持新的页面路径

### 自定义样式
可以通过修改组件中的 scoped CSS 或全局样式文件来自定义外观。

### 集成后端
可以将 localStorage 替换为 API 调用，实现数据的云端存储和同步。
